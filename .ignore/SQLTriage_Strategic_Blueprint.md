# 🧠 Governance & Business Value

## Turn SQL Server Monitoring into Business Control

SQLTriage goes beyond performance diagnostics to provide **visibility, control, and audit readiness** across your SQL Server environment.

## 🔒 Compliance & Audit Readiness
In enterprise environments, audits are not optional.

SQLTriage helps teams stay prepared by:
- Detecting **configuration drift** from approved standards
- Identifying **security risks and excessive permissions**
- Highlighting deviations from **SQL Server best practices**
- Generating structured reports for internal and external audits

These insights support environments aligned with frameworks such as **SOC2, HIPAA, and ISO 27001**.

## 💰 Cost Management & Resource Efficiency
SQL Server is often one of the highest-cost components in the data stack.

SQLTriage helps reduce unnecessary spend by:
- Identifying **underutilized or over-provisioned servers**
- Highlighting inefficient queries driving excessive CPU or I/O
- Reducing the need for premature hardware or cloud scaling
- Supporting **right-sizing of infrastructure**

## ⚠️ Prevent the Cost of Downtime
Unplanned outages are one of the most expensive risks in IT operations.

**Cost = Downtime × Business Impact**

SQLTriage helps prevent outages by detecting:
- Blocking and concurrency issues
- Resource bottlenecks
- Configuration risks (memory, tempdb, file growth)
- Query plan regressions

Before they escalate into production incidents.

## ⚙️ Built for Low Risk Deployment
- No installation
- No agents
- No additional attack surface

SQLTriage runs safely in production environments with minimal overhead.

## 📊 The Outcome
- Faster audits
- Lower infrastructure costs
- Reduced operational risk
- Increased confidence in SQL Server performance

## 💰 3. Cost & Efficiency Insight Layer

**Purpose:** Provide the IT Manager with the data needed to justify budget or reduce waste.

- **Infrastructure Efficiency:** Identify instances where CPU utilization is consistently low (<10%), signaling potential **licensing over-spend**.
- **Performance Inefficiencies:** Missing indexes and implicit conversions are driving unnecessary I/O and CPU cycles.
- **Strategic Recommendation:** Optimize existing workloads to recover **15-20% of current resource capacity** before approving hardware or cloud scaling spend.

## 5. SOC2 Type II Certification Path (v2.0+ Goal)

### Current State (v1.0)
SQLTriage v1.0 is a **tool, not a certified service**. It provides evidence for customers' own SOC2 audits but is not itself SOC2-certified.

### Target (v2.0–v2.5): Support customers pursuing certification
When offering a hosted SaaS (multi-tenant) version, SQLTriage operator will need SOC2 certification. For on-premises v1.0, customers use SQLTriage as evidence of THEIR own controls.

#### Certification Readiness Checklist (SQLTriage as evidence provider — v1.0 covers these):
- [x] **Audit Trail Completeness** — Tamper-proof HMAC logging, user action tracking, exportable for 90+ days
- [x] **Access Control Evidence** — RBAC with role assignment logs; AD/LDAP integration (v1.5)
- [x] **Change Management Evidence** — Configuration drift detection, historical baselines
- [x] **System Monitoring Evidence** — Alert history, incident correlation, uptime tracking
- [x] **Risk Assessment Documentation** — Governance Report with risk ratings and business impact

#### To become "SOC2-ready" as a SaaS provider (v2.0+):
- [ ] Third-party penetration test report (annual)
- [ ] Internal controls documentation (your own org's SOC2 if offering hosted service)
- [ ] Vendor security questionnaire (VSA, SIG) completed
- [ ] Data processing agreement (DPA) available for customers
- [ ] Incident response plan documented and tested
- [ ] Business continuity / disaster recovery plan

**Strategic note:** You do NOT need SOC2 certification for the **desktop tool**. Your customers use SQLTriage as *evidence* for THEIR SOC2 audits. If you launch a **hosted SaaS** version (multi-tenant, cloud), then YOU need SOC2 certification as a service provider.

## 🔧 6. The "Top 10" Priority Action Plan

**Purpose:** Give the team a clear roadmap for the next 30 days.

1.  **Zero-Loss Backups:** Ensure 100% backup coverage for all user databases.
2.  **Critical Alerting:** Configure SQL Agent alerts for Severity 17-25 and Corruption events.
3.  **Security Hardening:** Review and remove "Dangerous Service Accounts" (LocalSystem/Admin).
4.  **Resource Baseline:** Set **Max Server Memory** and **Cost Threshold for Parallelism**.
5.  **Audit Logs:** Enable and review audit logging for all failed login attempts.
6.  **I/O Optimization:** Move System and User databases off the OS (C:) drive.
7.  **Plan Stability:** Enable **Query Store** across the estate for regression tracking.
8.  **TempDB Health:** Standardize TempDB file counts to match logical CPU cores (max 8).
9.  **Standardization:** Upgrade instances to the latest **Cumulative Update (CU)**.
10. **Waste Reduction:** Resolve the top 10 "Missing Index" findings to lower CPU overhead.
