## 3. IFindingTranslator Service — Core Architecture ✓

**Decision:** Implement `IFindingTranslator` as a dedicated service producing three audience-specific renderings per finding.

**Rationale:**
- Opus identified GovernanceService alone does NOT translate — this is the product's core differentiator
- Translation substrate is what saves clients $50M: DBA language → CFO language
- Each finding output = `{ dba, itManager, executive }` objects with tailored metrics, risk语言, and cost implications

**Scope within translator:**
- Finding → DBA: technical details, T-SQL snippets, wait stats, index recommendations
- Finding → IT Manager: SLA impact, resource planning, root cause category, effort estimate
- Finding → Executive: business risk, cost exposure, compliance flags, 1-sentence summary

**Tie to other decisions:**
- Governance scoring weights feed into `executive.riskScore`
- RBAC determines which findings are visible per audience
- AuditLog captures every translation rendered

## 4. Governance Scoring Redesign ✓

**Decision:** Implement capped scoring with per-framework vector and JSON-editable weights.

**Spec:**
- Critical failures (P1, Service Down) capped at 40 points each (max contribution)
- Per-framework vector: Security (30%), Performance (25%), Reliability (25%), Cost (15%), Compliance (5%)
- Weights stored in `Config/governance-weights.json` (reloadable)
- GovernanceService.Check() returns `GovernanceReport` with per-category scores + capped total
- Maturity map uses capped scores per QuickCheck subset

**Rationale:**
- Prevents single P1 from dominating scorecard
- Allows product to tune emphasis via config (no recompile)
- Opus noted original scoring was unbounded and misleading

## 5. RBAC — Argon2id Password Hashing ✓

**Decision:** Use Argon2id (not CredentialProtector AES encryption) for RBAC user passwords.

**Rationale:**
- CredentialProtector encrypts, doesn't hash — wrong primitive for passwords
- Argon2id is memory-hard, GPU-resistant, OWASP-recommended
- Store `salt` + `hash` + `parameters` in RBAC user record
- DPAPI-machine still protects stored secrets (connection strings), but passwords are Argon2id hashed

**Implementation:**
- `RbacService.ValidateCredentials()` uses Argon2id verify
- `RbacService.CreateUser()` derives hash with random salt
- Existing `CredentialProtector` retained for machine-level secret encryption only

## 6. AuditLog — Single-Writer Queue + Checkpoint + Event Log Mirror ✓

**Decision:** Implement `AuditLogService` with single-writer queue, DPAPI-machine encryption, 4 KB checkpoint files, and Windows Event Log mirror for forensic integrity.

**Rationale:**
- Race conditions possible with concurrent logging in WPF app
- Checkpoint files allow recovery after crash
- Event Log provides Windows-integrated forensic trail (cannot be tampered from within app)
- DPAPI-machine ensures secrets inaccessible outside this machine

**Design:**
- `BlockingCollection<AuditEvent>` queue (single consumer Task)
- Flush to `%PROGRAMDATA%\SQLTriage\audit\{date}.log.enc` every 50 events or 5 s
- Also write plain-text summary to Windows Application Event Log (Source: "SQLTriage-Audit")
- On startup, replay partial checkpoint from last complete record

## 7. QuestPDF for Reports ✓

**Decision:** Use QuestPDF 2024.x for PDF report generation.

**Rationale:**
- Fluent API better than iTextSharp licensing complexity
- Good Blazor integration via QuestPDF.Previewer (dev-time)
- Supports headers/footers, tables, charts (via image snapshots)
- MIT-licensed, actively maintained

**Integration:**
- `ReportService.GenerateAssessmentPdf()` returns `byte[]`
- Pages: Cover, Executive Summary (translator), Findings (table), Governance Score, Recommendations
- Embedded charts via ApexChart `ToBase64Image()` helper

## 8. Error Catalog — ~60 Scenarios + Coverage Test ✓

**Decision:** Expand `ErrorCodes.cs` to ~60 distinct error scenarios with unit test.

**Catalog scope:**
- Connection failures (network, auth, timeout, version mismatch)
- Query timeouts / cancellation / deadlock victim
- Permission denied (SELECT/EXECUTE/VIEW SERVER STATE)
- Invalid credentials (SQL/Windows/Azure AD)
- Malformed responses / XML parse errors (system_health)
- Disk full / quota exceeded
- Azure Blob SAS failures (directory-scoped)
- Feature disabled by policy
- Concurrent modification conflicts
- Corrupt cache files

**Test:**
- `ErrorCatalogTests` enumerates all ErrorCodes, asserts non-zero Id, Value ≠ null, Description > 20 chars, Audience (DBA/IT/Exec) set

## 9. Quick Check — Curated Subset, Not Full VA ✓

**Decision:** Quick Check is a curated ~40-check subset tagged `"quick": true` in `queries.json`, runnable ≤60 s.

**Rationale:**
- Full 489-check VA takes 7–10 minutes — too slow for acquisition hook
- Curated subset covers 80% of common issues with 20% of runtime
- Tagging allows product to tune subset without code change
- Results feed Governance score as weighted subset

**UX:**
- Onboarding checkbox: "Enable Quick Check on startup (recommended)"
- Tray icon "Triage Now" → instant Quick Check → notification balloon with severity summary
- Full VA available from Dashboard → "Assess All 489 Checks" (explicit action)
