## ✅ Decisions Made (All Affirmative)

### 1. Adopt Option D — In-Place Hardening (NOT Option C Fresh Start)
- **Rationale:** Saves ~2 weeks, retains battle-tested code, avoids cherry-pick complexity, keeps git history.
- **Action:** Work from `v1.0-preparation` branch; rename/rebrand in-place; suppress CA1416; add new services incrementally.

### 2. Trim v1.0 Scope to Must-Have Only (13–15 tasks)
- **Cut to v1.1:** AI/ML (Task 23), Rolls Royce UI theme (Task 24), AD/LDAP (26), GPO templates (27), Documentation Generator (31), Code Signing (32).
- **Rationale:** 33 tasks in 8 weeks is impossible; pre-mortem correct; ship smaller faster.
- **New timeline:** 6 weeks (Weeks 1–6) with buffer for beta.

### 3. Add IFindingTranslator as Core Differentiator
- **Purpose:** Validates PRD Philosophy §1.0 — SQLTriage as "communication substrate" translating technical findings into DBA/IT/Executive renditions.
- **Implementation:** `Data/Services/FindingTranslator.cs` + `finding-translations.json` (seed ~30 high-severity checks). Integrated into `GovernanceService.GetRiskRegister()`.
- **Rationale:** Without this, the "translation layer" claim is marketing fluff. This is the soul of the project.

### 4. Fix RBAC Password Hashing — Use Argon2id (NOT CredentialProtector)
- **Change:** Replace reversible encryption with one-way Argon2id via `Konscious.Security.Cryptography.Argon2`.
- **Schema:** `PasswordHash` stores `$argon2id$v=19$m=65536,t=3,p=4$salt$hash`.
- **Rationale:** Security bug in current spec; compliance requires proper password hashing.

### 5. Redesign Governance Scoring with JSON Weights + Critical-Failure Cap
- **New algorithm:**
  1. Per-control PASS/WARN/FAIL (framework-tagged).
  2. Per-framework score = `(pass + 0.5*warn) / (pass+warn+faul)`.
  3. Overall = weighted mean of frameworks.
  4. **Critical cap:** any CRITICAL FAIL caps overall at 50%.
- **Weights stored in:** `Config/governance-weights.json` (editable, versioned).
- **Rationale:** Current additive model is broken; arbitrary base 70%; ignores critical single-point failures.

### 6. Use QuestPDF for Governance Report (NOT PdfSharpCore)
- **Library:** QuestPDF (MIT Community, commercial >$1M revenue).
- **Structure:** Component-based (`ExecutiveSummaryPage`, `ConfigDriftPage`, `ActionPlanPage`, optional `EvidenceAppendix`).
- **Sample generator:** `SQLTriage.exe --generate-sample-report` from `SampleFindings.json`.
- **Rationale:** Fluent API, testable, far less code than absolute positioning.

### 7. Implement AuditLogService with Single-Writer Queue + Checkpoint + Event Log Mirror
- **Design:** `Channel<AuditEntry>` background writer; DPAPI-Machine HMAC key with rotation window; `AuditLogCheckpoint` every 10K rows; simultaneous Windows Event Log write as tamper-evident anchor.
- **Rationale:** Prevents concurrent chain breaks; ensures startup validation is fast; provides true tamper evidence.

### 8. Define Quick Check as Curated Subset (~40 checks tagged "quick")
- **Not:** 472 checks. Must be ≤60s P95 on local SQL.
- **Tagging:** Add `"quick": true` to `queries.json` entries.
- **Cold-start:** Warm SQLite cache on splash screen before navigation.
- **Rationale:** 60s hook is impossible with full VA; targeted subset creates instant value.

### 9. Add Telemetry and Crash Reporting (1-day each, outsized value)
- **Telemetry:** Opt-in local `telemetry.jsonl` + "Upload" button (user-owned endpoint). Metrics: page views, Quick Check latency, export counts.
- **Crash dumps:** `AppDomain.UnhandledException` → `logs/crash-<timestamp>.log`; prompt on next launch to share.
- **Rationale:** Building in vacuum is pre-mortem's #2 risk. These mitigate.

### 10. Use ApexCharts (current) with ChartTheme Helper (Rolls Royce palette)
- **No MudBlazor/Syncfusion component libraries** — stay with current ApexCharts, add `ChartTheme.cs` for consistent styling (emerald/amber/slate, smooth bezier, dashed forecast, gradient confidence).
- **Tailwind:** Install for layout utilities only, not full rewrite.
- **Rationale:** Chart library change is unnecessary churn; ApexCharts already works; styling upgrade is sufficient.
