# Guide: Turning a Roast into Constructive Feedback

## Overview
This guide provides practical steps to transform harsh criticism (roasts) into friendly, constructive feedback. Useful for code reviews, documentation, or any feedback process.

## Steps

### 1. Strip or Soften Insult Elements
- Replace judgmental phrases with neutral observations.
- Turn condemnations into suggestions.
- Remove punitive ratings and vivid metaphors.

### 2. Shift to Feedback + Guidance
- Highlight what's working well.
- Provide actionable suggestions.
- Frame positively for improvement.

### 3. Use Neutral Language & Softened Adjectives
- "Spaghetti" → "complex"
- "Mess" → "area for improvement"
- "Naïve" → "simplistic"

### 4. Add a Positive Closing
- End with encouragement and balance.

### 5. Edit Document Structure
- Strengths → Opportunities for Improvement.
- Neutral headings like "Overview" and "Recommendations".
- Summary box stating feedback purpose.

### 6. Review for Tone
- Read aloud for friendly conversation feel.
- Check for sarcasm.
- Get peer review.

### 7. Example Rewrite
- Original: "Your architecture looks like a rusted Swiss-cheese..."
- Revised: "The current architecture presents several gaps that could affect reliability. Addressing these areas would significantly improve stability."

## TL;DR Checklist
- Replace insults with observations.
- Add positives before criticism.
- Use soft language.
- Structure into strengths/improvements.
- Encourage at end.
- Proofread for tone.

This guide helps maintain substance while making communication collegial.
